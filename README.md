# RepoGallery

A sleek, GitHub-styled gallery page to showcase your repositories. Drop in your repo URLs, run one command, and publish via GitHub Pages.

![screenshot](https://img.shields.io/badge/demo-live-blue?style=flat-square)

## ✨ Features

- **GitHub-native UI** — Clean card layout with stars, language dots, topic tags
- **Full README in a modal** — Click the excerpt on any card to see the complete README rendered as formatted markdown
- **Sorting** — Sort by stars, name, or last updated; toggle ascending/descending
- **Responsive** — Works on desktop, tablet, and mobile
- **Zero runtime API calls** — Data is pre-fetched and statically served
- **One-command setup** — `bash fetch-repos.sh` generates all data from your URL list

## 🚀 Quick Start

### 1. Use this template

Click **"Use this template"** on GitHub, or clone and push your own repo.

### 2. Add your repositories

Edit `URLs.txt` with your GitHub repo URLs, one per line:

```
https://github.com/your-username/repo-one
https://github.com/your-username/repo-two
```

### 3. Fetch repo data

```bash
# Optional: set a GitHub token to avoid rate limits
export GITHUB_TOKEN=ghp_your_token_here

bash fetch-repos.sh
```

This generates `repos.json` with all metadata + full README content for every repo.

### 4. Configure sorting (optional)

Edit `config.json`:

```json
{
  "sortBy": "stars",
  "sortOrder": "desc"
}
```

`sortBy`: `"stars"` | `"name"` | `"updatedAt"`  
`sortOrder`: `"asc"` | `"desc"`

### 5. Build

```bash
npm install
npm run build
```

### 6. Deploy to GitHub Pages

Push to `main` → CI automatically builds and deploys to the `gh-pages` branch.

**One-time setup in your repo settings:**
1. Go to **Settings → Pages**
2. Source: **Deploy from a branch**
3. Branch: **`gh-pages`**, folder: **`/root`**
4. Save

Or manually serve locally:

```bash
python3 -m http.server 3000
open http://localhost:3000
```

## 📁 Project Structure

```
├── index.html            # Entry page (GitHub-styled CSS inlined)
├── src/
│   ├── index.ts          # App entry — loads data, renders, wires up UI
│   ├── gallery.ts        # Card rendering, sorting, README modal
│   └── types.ts          # TypeScript interfaces
├── config.json           # Sort preference
├── repos.json            # Fetched repo data (generated by script)
├── URLs.txt              # Your repository URLs
├── fetch-repos.sh        # Data fetching script

├── tsconfig.json

```

## 🛠 Customization

### Change the title

Edit the header template in `src/index.ts`:

```ts
<a href="https://github.com/YOUR_USER" class="gallery-title-user">YOUR_USER</a>'s RepoGallery
```

### Change the accent color

The design uses GitHub blue (`#0969da`). To use your own color, replace all occurrences in `index.html`:

- `--color-accent: #0969da`
- `.card-readme` border-left
- `.lang-btn.active` background
- `.github-corner svg` fill

### Add filter/search (not included by default)

Filter/search was removed for simplicity. If you want it back, the original `gallery.ts` had `setupFilterUI()` and `filterRepos()` — check the git history.

## ⚙️ How it works

1. **`fetch-repos.sh`** reads `URLs.txt`, calls the GitHub API for each repo (metadata + README), and writes everything to `repos.json`
2. **`npm run build`** runs esbuild to compile `src/*.ts` into a single `dist/bundle.js`
3. **`index.html`** loads the bundle, which fetches `repos.json` and `config.json`, then renders the gallery
4. **GitHub Actions** (`.github/workflows/deploy.yml`) runs the build on every push and deploys to the `gh-pages` branch
